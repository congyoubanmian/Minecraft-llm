# 2026-05-28 世界重置与 LED 性能经验

## 背景

连续生成天宁宝塔、江南街区、广州塔等大型建筑后，世界里堆积了大量区块、schematic 粘贴结果和动态灯光实验。广州塔彩虹 LED 动画虽然视觉更接近夜景灯光秀，但在本机 2G Minecraft 内存配置和手机基岩版连接场景下，持续 `setblock` 刷新会明显拖慢服务端和客户端。

本轮目标是先把世界清空重启，恢复一个轻量的 Minecraft 环境，同时把大建筑和灯光方案里的经验沉淀下来。

## 结论

1. 大型建筑生成要把“世界状态”当成资源管理对象，不能无限往同一个世界堆项目。
2. 动态 LED 不适合默认开启。几百格高塔上每秒刷新数百到上千个方块，会压低 TPS，也会让低配电脑和手机端卡顿。
3. 对地标夜景，静态彩色玻璃加隐藏光源通常比动态刷方块更实用。远看效果稳定，性能成本接近普通建筑。
4. 每次重置世界前应先移动旧世界到 `backups/world_reset_时间戳/`，不要直接删除。
5. Git 只提交生成逻辑、DSL、前端和经验文档，不提交 `server/` 世界数据和 `backups/`。

## 有效做法

- 重置世界流程：
  - 先停止 `minecraft` 和 `bot` 容器。
  - 把 `server/world`、`server/world_nether`、`server/world_the_end` 移到 `backups/world_reset_YYYYMMDD_HHMMSS/`。
  - 保留 `server/plugins`、`server/config`、`server.properties` 和 schematic 目录。
  - 重新启动 `minecraft` 和 `bot`，让 Paper 自动生成干净世界。
- 广州塔 LED 方案：
  - 动态版用 datapack tick/function 做帧动画，不使用红石电路铺满塔身。
  - 默认应使用静态彩色帧，动画只作为可选调试模式。
  - 手机端优先保证可见、不卡、能连接，再考虑细节密度。
- 性能判断：
  - 观察 `tps`、FAWE 高内存告警、客户端视距加载情况。
  - 如果 TPS 低于 18 或客户端明显卡顿，应降低动画频率、减少每帧 `setblock` 数量，或者直接改静态灯光。
  - 超高建筑不要和大面积街区堆在同一个测试世界里长期保留。

## 下次优化

- 给项目列表增加“归档/清理世界”入口，让世界重置从手工命令变成后台操作。
- 把建筑生成分成三档：
  - `preview`：少量方块，前端快速看轮廓。
  - `static_build`：正式 schematic，静态灯光和细节。
  - `animated_show`：额外 datapack，可手动启用，默认关闭。
- 生成器应输出性能预算，例如预计方块数、实体数、每 tick 命令数、建议视距和最低内存。
- 动态灯光后续优先改成“局部动画”，只在观景台、塔顶或少量屏幕区域动，不让整塔持续刷新。
