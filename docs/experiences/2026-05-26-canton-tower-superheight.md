# 2026-05-26 广州塔超高版经验

## 背景

广州塔一开始沿用了偏“宝塔/分层楼阁”的思路，结果比例和轮廓都不对。后面改成了超高世界里的广州塔专用模板，并要求夜景有明显的 LED 立面和灯光设备。

## 结论

1. 广州塔不能用 pagoda 逻辑建，必须单独走 `twisted_lattice_tower` 语义模板。
2. 这类地标的关键不是堆方块数量，而是保住轮廓、腰身、观景层和塔顶天线的比例。
3. 夜景“像不像”，比单纯增加亮度更重要。LED 屏幕要沿塔身曲率铺，不要做成矩形贴片。
4. 超高世界是必要条件，但顶高要控制在 Bedrock 安全范围内。
5. 先生成 schematic 再进游戏验证，比只看 preview 更可靠。

## 有效做法

- 先把建筑拆成设计规约，再转 DSL。
- 把塔身分成结构层、观景层、灯光层、天线层。
- 灯光层不要只放点状光源，要沿外壳铺连续的发光壳。
- 观景检查要在 `midnight + clear` 条件下看，白天容易误判。
- 出生点和观景点要放在塔外，否则容易卡在结构里。
- 粘贴后第一件事是查日志，确认 `//paste -a` 真执行了。

## 下次优化

- 把重复结构继续抽成组件，比如塔身外壳、观景层、灯带环、塔顶模块。
- 对不同建筑类型分模板，不要让现代塔楼逻辑污染古建。
- 给每次项目补一个经验文件，和 `plan.json` / `state.json` 一起留档。
- 后续可以把这些经验再汇总成模板库，直接喂给 LLM 规划器。
